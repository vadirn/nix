Пулы соединений с базой данных — это концепция, которая помогает оптимизировать работу приложений. Создание нового соединения требует времени: TCP-хэндшейк, согласование TLS и аутентификацию. Повторное использование существующих соединений устраняет эти накладные расходы. Пул поддерживает набор простаивающих соединений. Когда запрос нуждается в соединении, он берёт его из пула. По завершении запроса соединение возвращается. Размер пула по умолчанию — 10 соединений. Максимальный размер — 50. Таймаут простоя — 30 секунд. Эти значения нужно настроить под вашу нагрузку.
