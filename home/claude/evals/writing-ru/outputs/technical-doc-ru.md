Пул соединений позволяет повторно использовать соединения с базой данных вместо создания новых для каждого запроса. Новое соединение — дорогая операция: оно требует TCP-хэндшейка, согласования TLS и аутентификации. Повторное использование соединений устраняет эти накладные расходы.

Пул поддерживает набор простаивающих соединений. Когда запрос нуждается в соединении, он берёт его из пула. По завершении запроса соединение возвращается.

Размер пула по умолчанию — 10 соединений. Максимальный размер — 50. Таймаут простоя — 30 секунд. Настройте эти значения под свою нагрузку.
